# 受众级别说明

`generate_handoff.py --client-level <level>` 支持三种受众级别，影响文档的写作风格和详略程度。

## non-technical（非技术受众）

**典型读者**：项目经理、产品经理、客户方对接人

**写作要求**：
- 避免使用技术术语，必须用时给出一句话解释
- 每个操作步骤给出**截图位置**或**精确点击路径**（如"点击左侧菜单栏的 Settings → 选 Environment Variables"）
- 部署文档用"在网页上点什么"代替命令行操作
- 架构图用简化的示意图，不标技术组件名
- 环境变量只写"需要配置哪些密码和地址"，不写变量名

**适用场景**：外包交付给非技术客户、交接给项目经理

## developer（开发者受众）— 默认

**典型读者**：接手项目的开发者、AI agent

**写作要求**：
- 使用标准技术术语，必要时简要说明
- 操作步骤用**可复制粘贴的命令**
- 架构图用 mermaid，标出技术组件和数据流
- 环境变量写变量名、格式、用途
- 代码引用写 `文件路径:行号`

**适用场景**：团队内部交接、开源项目文档、AI agent 接手

## devops（运维受众）

**典型读者**：运维工程师、SRE、平台工程师

**写作要求**：
- 重点突出**部署流程、监控告警、扩容策略、故障恢复**
- 命令包含完整的 flag 和参数
- 写明资源规格（CPU/内存/磁盘）、端口映射、网络配置
- CI/CD 写明完整的 pipeline 配置和 secret 清单
- Runbook 中的故障处置写明**排查命令和预期输出**

**适用场景**：交接给运维团队、生产环境部署文档
