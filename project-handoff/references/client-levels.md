# Client Levels

Use the correct client level before generating the package.

## non-technical

Use plain language.
Focus on operating the project, updating content, billing-sensitive systems, credentials ownership, and maintenance schedules.

## developer

Include repository structure, environment variables, local setup, build steps, API assumptions, and extension points.

## devops

Include infrastructure layout, deployment flow, CI/CD, secrets handling, monitoring, rollback, backups, and scaling notes.
